What is an end system (networking)?	A device at the edge of a network that sends or receives data, such as a computer, phone, printer, or sensor.
What is an intermediate system (networking)?	A device inside the network that forwards or relays traffic, such as a router, switch, or modem.
What is a gateway in networking?	An intermediate system that translates, filters, or connects between different networks or protocols.
What is a network topology?	The arrangement of nodes and links in a network.
What is a point-to-point channel (networking)?	A communication link where data is sent between exactly two endpoints.
What is a broadcasting channel (networking)?	A shared communication medium where one transmission can potentially be heard by many nodes.
Why can broadcasting channels suffer collisions?	If two nodes send at the same time, their signals can interfere and ruin both messages, so shared-media networks need coordination and error detection.
Why are layered network architectures used?	To manage complexity by splitting communication tasks into layers with clearer responsibilities.
What is encapsulation in networking?	The process where each layer wraps higher-layer data with its own control information before sending it downward.
What does the physical layer provide?	A bit stream between adjacent systems, including the mechanics of encoding and decoding bits.
What does the data link layer provide?	A frame stream between adjacent systems, including framing, medium access, and link-level error handling.
What is framing in the data link layer?	Grouping a bit stream into frames with boundaries and control information such as checksums.
What address is commonly used at the data link layer?	A MAC address.
What does the network layer provide?	Packet transport between end systems, including addressing and routing.
What is routing?	Choosing a path through the network from a source node to a destination node.
What are common responsibilities of the network layer?	Addressing, routing, choosing paths, handling congestion, and multiplexing transport-layer packets.
What does the transport layer provide?	End-to-end communication between processes.
What does the transport layer address that the network layer does not?	Processes on end systems, not just the end systems themselves.
What transport-layer service choices can vary between protocols?	Connection setup, ordering, error correction, and flow control.
What is end-to-end flow control?	Adjusting sending speed so a sender does not overwhelm a receiver.
What does the session layer handle?	Structured dialogue over time, such as session management, synchronization, checkpointing, and token management.
What does the presentation layer handle?	Data representation and semantics, such as encoding, serialization, compression, and encryption.
Name the 7 OSI layers from top to bottom.	Application; Presentation; Session; Transport; Network; Data link; Physical.
How does the TCP/IP model differ from the OSI model?	It merges application, presentation, and session into one application layer, and merges data link and physical into a network interface layer.
Why are the upper layers less clearly separated in TCP/IP?	Session and presentation functions are mostly treated as application support rather than distinct Internet-wide layers.
Why can the boundary between transport and network be blurry in TCP/IP?	Some mechanisms, such as congestion signaling, involve both layers.
What is the Internet hourglass model?	A protocol-stack structure with many applications above and many link technologies below, connected through a narrow common Internet layer.
Why is IP considered the waist of the Internet hourglass?	Because many different applications and network technologies interoperate through the common IP layer.
What is an N-entity in layered networking?	An active component within layer N that implements the layer's behavior, such as a process or intelligent I/O module.
What is an N-service access point (SAP) in layered networking?	A named interface where layer N provides service to layer N+1.
What is the relationship between peer entities in networking?	They are corresponding entities in the same layer on different systems that communicate using that layer's protocol.
What does a communication protocol define (networking)?	The format and ordering of messages between entities, and the actions taken on transmission, reception, or other events.
What is a protocol data unit (PDU)?	A unit of data exchanged by peer entities in a protocol; depending on context it may be called a frame, packet, datagram, or message.
What happens to user data as it moves down a protocol stack?	Each layer adds its own control information for its peer before passing the result downward.
Why does a receiving layer remove its own header before passing data upward?	Because that header is only meant for the peer entity at the same layer, not for higher layers.
What does a transport-layer port identify?	A process or application endpoint on an end system.
Why is a port called a transport-layer address?	Because it names the destination process within the host, which is the transport layer's addressing job.
What does a network-layer address identify?	The remote end system that packets should be delivered to across the network.
What does a link-layer MAC address identify?	The next-hop interface on the local link, not the final remote host across the whole path.
Why can the link-layer destination address change at each hop?	Because each hop sends the frame to a different next-hop interface while forwarding the same network-layer packet onward.
Why are MAC addresses unnecessary on a true point-to-point link?	Because there is only one possible peer on the link, so the lower layer already knows which interface should receive the frame.
Why are MAC addresses important on broadcast links?	Because multiple nodes share the medium, so frames need local link-layer addresses to identify the intended receiver.
What is address resolution in networking?	The process of mapping a network-layer address to the correct link-layer address for the next hop.
Why is address resolution needed in packet forwarding?	A node may know the destination IP address but still need the correct local MAC address before it can send a frame on the link.
Why is address resolution a per-link problem?	Different links may use different layer-2 protocols and address formats, so each hop needs its own local mapping.
Why can't a node on one side of a router use a host's MAC address from another link directly?	MAC addresses only have local link meaning and are not used end-to-end across routers.
What is network byte order?	The standard byte order used for multi-byte protocol fields on the network: big-endian.
What is the difference between big-endian and little-endian?	Big-endian stores the most significant byte first; little-endian stores the least significant byte first.
Why is a standard network byte order necessary?	So machines with different internal byte orders interpret transmitted multi-byte values consistently.
Why can raw memory views of integers be dangerous in networking code?	The byte order in memory depends on the host architecture, so blindly reinterpreting bytes can reverse protocol field values.
Why is big-endian convenient for protocol headers and addresses?	The most significant bytes come first on the wire, which matches prefix-style interpretation and lets receivers start processing from the leading bytes immediately.
What does CIDR allow in IP addressing?	Variable-length network prefixes, so networks can choose how many bits belong to the network part versus the host part.
What does a netmask indicate in IP addressing?	Which bits belong to the network prefix and which belong to the host portion of the address.
How is a CIDR network commonly written?	As a network address followed by a prefix length, such as `/22` or `/48`.
How can a host or router tell whether a destination is on a local subnet?	By checking whether `destination_address & netmask` equals the local subnet address.
What is the longest-prefix-match rule in IP routing?	If multiple routing entries match a destination, use the one with the longest matching prefix.
Why is longest-prefix match needed in routing?	So more specific routes override broader aggregate routes when both match the same destination.
Why was IPv6 introduced?	To address IPv4 limitations, especially address exhaustion, and to improve areas such as routing scalability, mobility, and QoS support.
How does the IPv6 header differ from the IPv4 header?	It is larger but simpler, with a fixed basic format and less per-hop processing.
Why is the IPv6 header considered simpler than the IPv4 header?	It removes the header checksum, moves options into extension headers, and eliminates router-side fragmentation.
Who handles fragmentation in IPv6?	End systems handle it; routers do not fragment packets in transit.
What is the purpose of IPv6 extension headers?	To move optional features out of the main header so normal forwarding stays simpler.
What IPv6 fields help support differentiated handling of traffic?	The traffic class and flow label fields.
How are IPv6 addresses written?	As eight groups of hexadecimal 16-bit values separated by colons.
How can an IPv6 address be compressed in text form?	By omitting leading zeros in a group and replacing one contiguous run of all-zero groups with `::`.
What does an IPv6 prefix like `/48` mean?	The first 48 bits are the network prefix, and the remaining bits identify subnets and interfaces.
What is a link-local IPv6 address?	A non-routable address on the local link, typically in `fe80::/10`, used for local communication and bootstrapping.
Why is a link-local IPv6 address useful even before a host gets a routable address?	It lets the host communicate on the local link first, which helps with setup steps such as duplicate-address detection and later address acquisition.
What message is used in IPv6 duplicate-address detection?	A Neighbor Solicitation ICMP message.
What does the transport layer address in the TCP/IP model?	Process or application endpoints, not just hosts.
What identifies a transport-layer service endpoint in TCP/IP?	A port number together with the host's IP address and transport protocol.
What uniquely identifies a TCP connection?	The 5-tuple of source IP, source port, destination IP, destination port, and protocol.
Why can multiple TCP connections reuse the same local port?	Because the full 5-tuple is different when the remote endpoint differs.
Why are well-known server ports useful?	They give clients a predictable port number for reaching a standard service.
What is transport-layer multiplexing and demultiplexing?	Using ports and protocol identifiers so many application flows can share one host and IP stack while incoming data is delivered to the correct process.
What information identifies the forwarding target in a connectionless network-layer packet?	The destination end-system address.
What information can identify the forwarding target in a connection-oriented network layer?	A route label or virtual-circuit identifier carried in the packet.
What knowledge does an intermediate system need for layer-3 forwarding?	Subnetwork topology, end-system location/addressing, communication requirements such as QoS, and current network status.
What is the difference between routing and switching in packet networks?	Routing computes the next hop from a destination address; switching forwards using a prefilled mapping from a short identifier to the next hop.
What is circuit switching?	Communication over a dedicated physical connection that exists for the duration of the conversation.
Why can circuit switching waste resources?	Capacity stays reserved even when the traffic is bursty or temporarily idle.
What is packet switching?	Sending limited-size packets independently through the network without a dedicated end-to-end path.
Why can packet switching achieve high resource utilization?	Links are shared statistically instead of being rigidly reserved for one conversation.
What are key drawbacks of packet switching?	Congestion can occur, delay can vary, and packets from one flow may take different paths.
What is virtual-circuit switching?	Connection-oriented packet switching in which a path is set up in advance and switches keep per-connection state.
How does virtual-circuit switching differ from circuit switching?	It still sends packets and keeps logical state, but it does not require a dedicated physical circuit for the whole path.
What happens during the setup phase of a virtual circuit?	A path is selected, virtual-circuit identifiers are assigned, forwarding state is stored in switches, and resources may be reserved.
How are packets forwarded during a virtual circuit's data-transfer phase?	All packets follow the established path using the stored forwarding state and the virtual-circuit identifier in each packet.
What happens during the disconnect phase of a virtual circuit?	The switches release reserved resources and delete the connection state.
Why can virtual-circuit packets have lower header overhead than pure packet-switched packets?	They can carry a short virtual-circuit identifier instead of a full destination address.
What benefits can virtual circuits provide over pure packet switching?	They can help preserve packet order and make QoS guarantees more feasible.
Why are virtual-circuit identifiers often local to each link segment instead of global?	The same numeric identifier can be reused on different links, and each switch maps incoming identifiers to outgoing ones.
Why is virtual-circuit switching more sensitive to node or link failures than packet switching?	Its fixed path and stored per-connection state depend on the specific switches and links chosen during setup.
What are the core responsibilities of the transport layer in TCP/IP?	Addressing processes, managing end-to-end connections, transferring data transparently between processes, and offering QoS-related features such as reliability and flow/congestion control.
What is a connection-oriented transport service?	A transport service with separate setup, data-transfer, and disconnect phases.
What is a connectionless transport service?	A transport service that sends independent messages without first establishing a connection.
What is the difference between flow control and congestion control?	Flow control protects the receiver from being overwhelmed; congestion control protects the network from overload.
Why can flow control and congestion control be hard to distinguish in practice?	Both can appear as packet loss or reduced throughput, even though one is caused by receiver limits and the other by network limits.
What is a transport service access point at the transport layer in TCP/IP?	A port together with the host's IP address.
What is a transport protocol data unit (TPDU)?	The unit of data exchanged by transport-layer peers; in TCP/IP it is commonly called a segment or datagram depending on the protocol.
Why is TCP described as a byte-stream service rather than a packet service?	It presents applications with an ordered stream of bytes instead of preserving application message boundaries.
Why does TCP usually split data into segments that fit into layer-3 packets?	To avoid relying on IP fragmentation, which is inefficient and can make the loss of one IP fragment destroy the whole larger unit of data.
What kind of transport protocol is UDP?	A simple, unreliable, connectionless, message-oriented transport protocol.
What is the purpose of the UDP source port?	It identifies the sender's transport endpoint so a reply can be directed back there.
What is the purpose of the UDP destination port?	It identifies the receiving service or process for demultiplexing at the destination host.
What does the UDP checksum protect?	The UDP header and data, with help from a pseudoheader derived from IP-layer information.
What is included in the UDP pseudoheader for checksum calculation?	The source IP address, destination IP address, protocol number, and UDP length.
Why does UDP include IP-layer information in its checksum pseudoheader?	So the receiver can detect some cases of misdelivery to the wrong IP endpoint or wrong protocol context.
When is the UDP checksum considered unused in IPv4?	When the checksum field is all zeros.
Why is a computed UDP checksum of zero transmitted as `0xFFFF`?	Because an all-zero checksum field is reserved to mean that no checksum is being used.
Why is UDP often a good fit for simple request/response or real-time applications?	It avoids connection setup and retransmission delays, which is useful when timeliness matters more than occasional loss or duplication.
