Why must `match` expressions be exhaustive in Rust?	Every possible value must be handled so execution never reaches an unaccounted-for case.
Why can `let` use tuple or struct patterns?	A `let` binding can destructure a value and bind multiple names at once.
Why can chained `if let` and `else if` sometimes express logic more conveniently than `match`?	They can combine unrelated boolean conditions and pattern matches in one control-flow chain.
What does `while let` do in Rust?	It repeats a loop for as long as a value continues to match a pattern.
What is an irrefutable pattern?	A pattern that matches every possible value of the expected type.
What is a refutable pattern?	A pattern that can fail to match some possible values.
Which Rust contexts require irrefutable patterns?	`let` bindings, function parameters, and `for` loop patterns.
Which Rust contexts are designed for refutable patterns?	`match` arms, `if let`, `while let`, and `let...else`.
Why does `let Some(x) = value;` fail to compile?	`let` requires an irrefutable pattern, but `Some(x)` does not match `None`.
What happens if `let...else` uses an irrefutable pattern?	The compiler warns because the `else` branch can never run.
Why can a named variable pattern inside `match` unexpectedly shadow an outer variable?	Pattern bindings create new variables in the match arm's scope rather than comparing against existing ones.
What does `|` mean in a Rust pattern?	It creates an OR-pattern that matches if any of the alternatives match.
What does `1..=5` mean in a Rust pattern?	An inclusive range pattern that matches any value from 1 through 5.
Why are range patterns limited to numbers and `char` in Rust?	The compiler must be able to determine at compile time whether the range is valid and non-empty.
Can Rust patterns destructure nested enums, structs, and tuples in one step?	Yes; patterns can be nested to match and bind deeply structured values in a single expression.
What does `_` mean in a Rust pattern?	Match this part of the value but do not bind it.
What is the difference between `_` and `_name` in a Rust pattern?	`_` does not bind the value at all, while `_name` still binds it and can therefore move ownership.
Why does `if let Some(_s) = s` move `s` but `if let Some(_) = s` does not?	`_s` binds the inner value, but `_` only matches and ignores it without binding.
What does `..` do in a Rust pattern?	It matches and ignores the remaining unmatched parts of a value.
Why can `..` be used only once in a tuple pattern?	Using it more than once would make it ambiguous how many elements each `..` should absorb.
What is a match guard in Rust?	An extra `if` condition attached to a `match` arm that must also be true for the arm to match.
Why are match guards useful?	They express conditions that cannot be captured by pattern syntax alone.
How can a match guard help avoid variable-shadowing problems?	You can bind a new name in the pattern and compare it to an outer variable in the guard.
How does a match guard interact with an OR-pattern like `4 | 5 | 6 if cond`?	The guard applies to the entire OR-pattern, not just the last alternative.
What does `@` do in a Rust pattern?	It binds a matched value to a variable while also testing that value against another subpattern.
Why is `id @ 3..=7` useful in a Rust pattern?	It both checks that `id` is in the range and keeps the matched value available for the arm's code.
