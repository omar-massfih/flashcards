What does `async def` define in Python?	A native coroutine function; calling it produces a coroutine object.
What is an asynchronous generator in Python?	A function defined with `async def` that uses `yield`, producing an asynchronous iterator.
What makes an `async def` function an asynchronous generator instead of a native coroutine?	It contains `yield` in its body.
What can `await` be applied to in Python?	Awaitable objects, such as native coroutine objects, `Task` objects, and other objects implementing the await protocol.
What is the practical difference between `await coro()` and `asyncio.create_task(coro())`?	`await` runs the coroutine and waits for its result now; `create_task` schedules it to run concurrently and returns a `Task`.
What does `asyncio.gather(*aws)` do?	It waits for all given awaitables to finish and returns their results in submission order.
How does `asyncio.gather` order its results?	It returns results in the same order the awaitables were passed in, not completion order.
When is `asyncio.as_completed` preferable to `asyncio.gather`?	When you want to handle results as each awaitable finishes instead of waiting for all of them together.
What does `asyncio.as_completed` yield?	Awaitables that produce results in completion order.
Why can't you directly reuse blocking I/O functions in high-performance asyncio code?	Because blocking functions stall the event loop instead of yielding control while they wait.
What is the all-or-nothing problem in asynchronous programming?	To get the full benefit, every I/O-heavy layer must avoid blocking; one blocking layer can undermine the whole async design.
What is an asynchronous context manager in Python?	An object used with `async with` that defines asynchronous `__aenter__` and `__aexit__` methods.
When should `async with` be used instead of `with`?	When entering or exiting the context requires awaitable setup or cleanup.
What is an asynchronous iterable in Python?	An object usable with `async for` because it provides `__aiter__`.
What is an asynchronous iterator in Python?	An asynchronous iterable that provides `__anext__`, returning an awaitable for the next item.
Why are asynchronous iterators useful for large or slow data sources?	They can yield control between items instead of blocking while waiting for more data.
What is an asynchronous generator expression in Python?	A generator expression using `async for`, producing an asynchronous generator object.
What does `asyncio.to_thread(fn, *args, **kwargs)` do?	It runs a blocking callable in a worker thread and returns an awaitable for its result.
Why is `asyncio.to_thread` useful in async programs?	It lets you offload blocking operations such as file I/O without freezing the event loop.
What does `loop.run_in_executor(executor, fn, *args)` do?	It submits a blocking callable to an executor and lets async code await its completion.
When might `run_in_executor` use a `ProcessPoolExecutor` instead of the default thread pool?	When the delegated work is CPU-bound and you want to avoid blocking on the GIL.
Why can cancellation be tricky with `run_in_executor`?	Cancelling the awaiting coroutine does not necessarily stop the underlying thread or process job.
What is an `asyncio.Semaphore` used for?	Limiting how many coroutines may enter a protected section concurrently.
Why are semaphores useful in async network clients?	They throttle concurrency so the client does not overwhelm a server or local resources.
What does `async with semaphore:` do in asyncio?	It acquires the semaphore asynchronously on entry and releases it automatically on exit.
Why is `async with semaphore:` safer than manual `acquire()` and `release()` calls?	It guarantees release even if an exception occurs.
Why should blocking file I/O inside an async program often be delegated to a thread?	Because file operations can block the event loop just like network calls.
What is structured concurrency in async programming?	A style where spawned concurrent tasks are scoped to a block or task group with a clear lifetime and cleanup point.
Why is a so-called I/O-bound async system still vulnerable to CPU-bound slowdowns?	Because any CPU-heavy code running in the event-loop thread can still block all other async tasks.
What are common ways to handle CPU-bound work in an async Python system?	Use a process pool, an external task queue, or compiled/native code instead of running it in the event-loop thread.
