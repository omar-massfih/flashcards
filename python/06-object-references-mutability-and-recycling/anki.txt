Python variables: are they “boxes” or “labels”?	Labels (names bound to objects), not boxes that contain copies.
If b = a and a refers to a mutable object, what happens to b when you mutate through a?	b reflects the same change because both names reference the same object.
Why can `y = Gizmo() * 10` create an instance but still not bind `y`?	The RHS creates the object, then an error occurs before binding; `y` is never created.
Why is `is` typically faster than `==` (Python)?	`is` can’t be overloaded; it’s essentially an identity comparison.
When is `is` the right tool in Python code?	Comparing to singletons like `None` or a sentinel object.
Idiom: how do you test for `None` in Python?	`x is None` (and `x is not None`).
How do you create a sentinel object for identity checks (Python)?	`SENTINEL = object()` and compare with `is`.
Common pitfall: when is using `is` usually wrong?	For value equality like strings/integers/collections; use `==` instead.
What does tuple immutability actually guarantee (Python)?	The tuple’s stored references don’t change; referenced mutable objects can still mutate.
How can a tuple “value” appear to change even if the tuple is immutable?	If it references a mutable item that changes in place.
How do you make a shallow copy of a list (common ways)?	`list(x)`, `x[:]`, or `x.copy()`.
In a shallow copy, what happens if you mutate a shared inner list?	The mutation is visible through both the original and the shallow copy.
How do you make shallow vs deep copies of arbitrary objects (Python)?	`copy.copy(x)` vs `copy.deepcopy(x)`.
Why can `deepcopy` handle cyclic references safely (Python)?	It memoizes objects already copied to avoid infinite recursion.
Why can a deep copy be undesirable (Python)?	It may be too expensive or may copy things that shouldn’t be copied (resources/singletons).
How can a class customize copying behavior (Python)?	Implement `__copy__` and `__deepcopy__`.
What is Python’s parameter passing model called?	Call by sharing.
Under call by sharing, what does a function receive for each argument?	A copy of the reference to the same object.
What can a function do to a mutable argument under call by sharing?	Mutate the object, affecting the caller’s view.
What can a function not do to the caller’s bindings under call by sharing?	It can’t rebind the caller’s name to a different object.
Mutating vs rebinding: what’s the difference inside a function (Python)?	Mutation affects the shared object; rebinding only changes the local parameter name.
Augmented assignment (`+=`): what’s the high-level rule for mutables vs immutables?	Mutables often change in place; immutables create a new object and rebind.
Why can `a += b` differ for lists vs tuples (Python)?	Lists can mutate in place; tuples are immutable so a new tuple is created.
When are default parameter values evaluated (Python)?	Once at function definition time (typically module import).
Where are positional default parameter values stored (Python)?	On the function object (e.g., `func.__defaults__`).
Why are mutable default arguments dangerous (Python)?	Mutations persist and affect future calls that use the default.
Safe pattern for optional mutable parameters (Python)?	Default to `None`, then create a new list/dict inside the function.
Why use `if x is None:` for optional parameters (Python)?	Distinguishes “argument omitted” from an empty container provided by the caller.
If a default list is mutated, who observes that mutation later?	Any future call that relies on that default object.
How do you defensively create your own list from an iterable argument?	`list(iterable)`.
Why is `list(iterable)` a flexible defensive-copy pattern?	It accepts any iterable and ensures you own a mutable list.
What does `del name` actually delete (Python)?	The reference/binding, not the underlying object.
When can an object be garbage-collected (Python, high level)?	When it becomes unreachable (no strong references).
In CPython, what primary mechanism destroys most objects promptly?	Reference counting (plus a cycle detector for cyclic garbage).
What problem does cyclic garbage collection address (CPython)?	Unreachable reference cycles that reference counting alone can’t free.
Is destruction timing guaranteed across Python implementations?	No; you should not rely on immediate finalization.
What is `__del__` for (Python)?	A finalizer hook when an object is about to be destroyed; tricky and rarely needed.
Why is relying on `__del__`/GC timing risky?	Finalization may be delayed or behave differently across implementations/cycles.
What is a weak reference (Python)?	A reference that doesn’t keep the target alive (doesn’t increase refcount).
What does `weakref.finalize(obj, callback)` do (Python)?	Registers `callback` to run when `obj` is garbage-collected, without keeping `obj` alive.
What is interning (Python, high level)?	Reusing identical immutable objects for efficiency.
Why shouldn’t you rely on interning for program logic?	It’s an implementation detail; use `==` for value comparisons.
