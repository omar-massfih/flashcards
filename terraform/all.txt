What is Infrastructure as Code (IaC)?	Managing infrastructure using configuration files instead of manual GUI/point-and-click changes.
Why is IaC “safer” than manual changes (in principle)?	Changes are explicit, reviewable, and reproducible from versioned config.
What is Terraform (in one sentence)?	HashiCorp’s IaC tool that defines infrastructure in declarative config files and manages its lifecycle.
What does “declarative configuration” mean (Terraform)?	You describe the desired end-state; Terraform determines the actions to reach it.
Terraform: what is a provider?	A plugin that lets Terraform interact with a platform/service via its API.
Terraform: what is a resource (conceptually)?	An individual unit of infrastructure managed by Terraform (e.g., compute instance, network).
Terraform: what is a module?	A reusable Terraform configuration composed from resources (possibly across providers).
How does Terraform handle dependencies between resources?	It automatically calculates dependencies to create/destroy in the correct order.
Terraform workflow: what are the core steps?	Scope; Author; Initialize; Plan; Apply.
Terraform workflow: what does “Initialize” do?	Install the plugins/providers Terraform needs to manage infrastructure.
Terraform workflow: what does “Plan” do?	Preview the changes Terraform will make to match the configuration.
Terraform workflow: what does “Apply” do?	Make the planned changes to reach the desired state.
What is Terraform state (purpose)?	A state file that tracks real infrastructure and acts as a source of truth.
How does Terraform use state during deployments?	It compares desired config to state to decide what changes to make.
Why is version control valuable with Terraform configurations?	It enables safe collaboration and change tracking via code review and history.
What is a remote state backend used for?	To securely share state and coordinate access for team collaboration.
What is HCP Terraform used for (high level)?	Hosted Terraform that can share state, run Terraform in a stable environment, and support collaboration.
How can Terraform integrate with VCS (high level)?	Connect to GitHub/GitLab/etc. to propose infra changes when config changes are committed.
What is the purpose of the `terraform { ... }` block?	Terraform settings like required providers and Terraform CLI version constraints.
What does a provider `source` address like `hashicorp/azurerm` specify?	Which provider to install (from the Terraform Registry by default).
Why add a provider version constraint in `required_providers`?	To avoid unplanned upgrades that may introduce breaking changes.
What does the Terraform version constraint `required_version` do?	Prevents using Terraform CLI versions that don’t satisfy the constraint.
What is a Terraform `provider` block used for?	Configure a provider plugin (credentials, features, regions, etc.).
What is a Terraform `resource` block used for?	Declare an infrastructure object Terraform will manage.
Resource block structure (Terraform): what are the two strings after `resource`?	Resource type and resource name (e.g., `resource "TYPE" "NAME"`).
What is a Terraform resource address (format)?	`<resource_type>.<resource_name>` (e.g., `azurerm_resource_group.rg`).
What does `terraform init` do?	Initializes the working directory (downloads providers/modules; sets up backend).
What does `terraform plan` do?	Shows a preview of changes needed to reach the desired state.
What does `terraform apply` do?	Applies the planned changes to create/update/destroy infrastructure.
What does `terraform fmt` do?	Formats `.tf` files to canonical Terraform style.
What does `terraform validate` do?	Checks configuration syntax and internal consistency (without applying).
What file does Terraform use to record deployed infrastructure state by default?	`terraform.tfstate`.
Why must you protect `terraform.tfstate`?	It can contain sensitive data in plaintext; don’t share or commit it.
What does `terraform state list` do?	Lists resource addresses tracked in the current state.
What does it mean to “change infrastructure” in Terraform?	Edit configuration, then run `terraform apply` to create/update resources to match the new desired state.
How do you add a new resource in Terraform (conceptually)?	Add a new `resource "TYPE" "NAME" { ... }` block to your configuration.
How do you reference another resource’s attribute in Terraform config?	Use `<TYPE>.<NAME>.<ATTRIBUTE>` (e.g., `azurerm_resource_group.rg.name`).
What does referencing `azurerm_resource_group.rg.name` accomplish in another resource?	Uses the existing resource’s value and creates an implicit dependency between the resources.
Why does Terraform create a dependency graph?	To ensure resources are created/updated in a valid order based on references.
In Terraform plan output, what does `+ create` indicate?	A new resource will be created.
In Terraform plan output, what does `~ update in-place` indicate?	An existing resource will be modified without replacing it.
What does it mean to “update in-place” (Terraform)?	The resource is modified while keeping the same identity/address; it is not destroyed and recreated.
In plan output, what does “(known after apply)” mean?	The value is computed by the provider during apply and isn’t known at plan time.
What does “Refreshing state...” mean during `terraform apply`?	Terraform is reading current real-world resource data to synchronize state before planning/applying changes.
Why can adding a reference create an implicit dependency without `depends_on`?	Because Terraform can infer ordering from the expression graph (data flow) in configuration.
What does the `tags` block do on a Terraform resource (Azure example)?	Assigns metadata key/value tags to the resource.
In HCL, what is the type/shape of `tags = { ... }`?	A map/object of keys to values (e.g., `Environment = "..."`).
Why are tags useful in cloud infrastructure?	They help with organization, ownership, environment labeling, and cost/accounting policies.
How does Terraform show tag additions in a plan?	As an in-place update (`~`) with new keys/values added under `tags`.
What does `terraform show` help you verify after an apply?	It displays the current state values Terraform knows for managed resources (including updated tags).
Why might Terraform show `id = (known after apply)` during planning?	IDs are assigned by the provider/cloud when the resource is created.
What does `terraform destroy` do?	Terminates (destroys) the resources managed by the current Terraform project/state.
Why destroy infrastructure when you no longer need it?	To reduce security exposure and costs (e.g., remove short-lived test/build environments).
Does `terraform destroy` delete resources not managed by the current project?	No; it targets only the infrastructure tracked/managed by the current Terraform state.
What does `terraform destroy` use to decide what to terminate?	The current state (local `terraform.tfstate` or remote backend state, depending on setup).
In destroy plan output, what does `- destroy` indicate?	That the resource instance will be destroyed.
In destroy plan output, what does `-> null` mean?	The attribute value will be removed because the resource is being destroyed.
What warning does Terraform give before destroying managed infrastructure?	There is no undo; only `yes` is accepted to confirm.
Does Terraform destroy resources in arbitrary order?	No; it destroys in an order that respects dependencies.
Why does destroy order matter (Terraform)?	Dependent resources must be destroyed before the resources they depend on (e.g., destroy VNet before resource group).
Why use input variables in Terraform?	To avoid hard-coded values and make configurations more flexible and reusable.
What is an input variable (Terraform)?	A named value you can set from outside the config (or via defaults) and reference as `var.<name>`.
Where do you commonly declare input variables (Terraform)?	In `.tf` files such as `variables.tf` using `variable "<name>" { ... }`.
What does `default = "..."` do in a Terraform variable?	Makes the variable optional by providing a value when no input is given.
When is a Terraform variable required input?	When it has no `default` and no value is provided via other mechanisms.
How do you reference an input variable in Terraform configuration?	Use `var.<variable_name>` (e.g., `var.resource_group_name`).
How do you override a variable value for a single run (Terraform)?	Pass it on the CLI, e.g. `terraform apply -var "resource_group_name=NewName"`.
If CLI `-var` values aren’t saved, how do you avoid retyping them?	Use other mechanisms like `.tfvars` files or environment variables.
What is a `.tfvars` file used for (Terraform)?	To provide variable values from a file so you don’t have to pass them on the CLI each time.
What environment-variable naming pattern can set Terraform variables?	`TF_VAR_<variable_name>` (e.g., `TF_VAR_resource_group_name`).
In plan output, what does `-/+` mean?	Destroy and then create replacement (resource will be replaced).
What does “must be replaced” mean in Terraform plan output?	The resource cannot be updated in-place for the change, so Terraform will replace it.
What does “forces replacement” indicate in plan output?	That changing the attribute requires destroying and recreating the resource.
Why can a dependent resource be replaced when its dependency is replaced?	Because it references attributes that change (e.g., `resource_group_name`), forcing it to be recreated too.
How does Terraform know which resources depend on others?	It infers dependencies from references like `azurerm_resource_group.rg.name` in other resources.
How do you tell replacement apart from in-place update in plan symbols?	`~` is update in-place; `-/+` is destroy then create replacement.
Why should you be cautious about putting secrets in variable defaults?	Defaults can end up in code/version control; prefer secret-safe inputs (env vars/secret managers).
What’s a typical variable definition “shape” you’ll see in real configs?	`variable "x" { type = string; description = "..."; default = "..." }`.
When should you add a `type` to a Terraform variable?	When you want stronger validation and clearer intent (e.g., `string`, `list(string)`).
Why use output values in Terraform?	To avoid digging through many resource attributes by exposing only the values you care about.
What is an output value (Terraform)?	A declared value (in an `output` block) that Terraform records and can display/query for the user.
When does Terraform print output values?	After `terraform apply` completes (it prints an “Outputs” section).
How do you query saved outputs from the CLI (Terraform)?	Run `terraform output`.
How do you query a single output by name (Terraform)?	`terraform output <output_name>` (e.g., `terraform output resource_group_id`).
Where do you typically define outputs (Terraform)?	In `.tf` files (often `outputs.tf`), using `output "..." { ... }`.
How do you reference a resource attribute in an output (Terraform)?	Use `<TYPE>.<NAME>.<ATTRIBUTE>` (e.g., `azurerm_resource_group.rg.id`).
After adding an output to an already-applied config, what must you run to see it?	Re-run `terraform apply`.
Can `terraform apply` save new outputs without changing resources?	Yes; the plan can be `0 to add, 0 to change, 0 to destroy` while outputs change.
Why are outputs useful in complex configurations?	They surface a few important values from hundreds/thousands of tracked attributes.
How does a parent configuration reference a child module’s output (Terraform)?	`module.<module_name>.<output_name>`.
What does `sensitive = true` do for an output (Terraform)?	Marks the output as sensitive so Terraform suppresses it in normal CLI output.
Does marking an output as sensitive keep it out of state (Terraform)?	No; it can still be stored in state, so you must protect state access.
What does `terraform output -json` provide?	Machine-readable JSON output for automation/scripts.
Why store Terraform state remotely (production)?	To keep state secure/encrypted and enable team collaboration via shared state access.
What is HCP Terraform (high level)?	A hosted Terraform platform for collaboration (versioning/auditing runs), remote execution, and remote state storage.
Why run Terraform in a remote environment (HCP Terraform)?	It provides shared access to state and a stable environment for long-running Terraform operations.
How do you configure HCP Terraform in Terraform CLI workflow (HCL block)?	Use a `terraform { cloud { organization = "..."; workspaces { name = "..." } } }` block.
What is an HCP Terraform workspace (conceptually)?	A named context that holds state and run history for a configuration.
What does `terraform login` do (HCP Terraform)?	Authenticates the CLI to HCP Terraform by obtaining/storing an API token.
Why should you protect `credentials.tfrc.json`?	It contains an API token; anyone with the file can access HCP Terraform as you.
What is the default execution mode for HCP Terraform CLI-driven workflow?	Remote execution (Terraform operations run in HCP Terraform).
What does “local execution” mean with HCP Terraform (CLI-driven)?	Terraform runs locally but stores state remotely in HCP Terraform.
Where should you set Azure `ARM_...` auth variables when using HCP Terraform?	In the HCP Terraform workspace Variables UI (not only as local shell exports).
Which Azure auth value should be marked sensitive in HCP Terraform variables?	`ARM_CLIENT_SECRET`.
What two main workflows does HCP Terraform support (names)?	VCS-driven workflow and API-driven workflow.
What triggers runs in an HCP Terraform VCS-driven workflow?	Commits/changes to the connected VCS repository queue plans/runs automatically.
Terraform language: what is HCL?	A configuration language Terraform’s native syntax is based on (also used by other HashiCorp tools).
Terraform config syntax: what are the two core constructs?	Arguments and blocks.
What is an argument in Terraform configuration syntax?	A name/value assignment like `image_id = "abc123"` where the value is an expression.
What is a block in Terraform configuration syntax?	A container with a block type, optional labels, and a `{ ... }` body containing arguments and nested blocks.
What are “block labels” (Terraform/HCL)?	Quoted strings after the block type keyword that identify a particular instance (label count depends on block type).
For a `resource` block, what are the two labels (high level)?	The resource type (provider-specific) and the resource’s local name (arbitrary within the module).
What is the purpose of the second label in `resource "TYPE" "NAME" { ... }`?	It uniquely names that instance so you can reference it (e.g., `TYPE.NAME`).
What is an identifier in Terraform configuration?	A name token used for argument names, block types, and Terraform constructs (resources, variables, etc.).
What characters can Terraform identifiers contain?	Letters; digits; underscores (`_`); hyphens (`-`).
What identifier rule avoids ambiguity with numbers?	The first character of an identifier must not be a digit.
Terraform resource meta-argument: what does `count` do?	Provisions multiple instances of the same resource from one block.
Terraform resource meta-argument: what does `for_each` do?	Provisions multiple instances from one block, one per element in a map or set.
What kinds of values can `for_each` take (Terraform)?	A map or a set of strings.
Terraform resource meta-argument: what does `depends_on` do?	Forces Terraform to complete operations on listed upstream references before operating on this resource.
Terraform resource meta-argument: what does `provider` do?	Selects a specific provider configuration (alias) to use for that resource.
What is the purpose of `lifecycle { ... }` on a resource (Terraform)?	Defines lifecycle rules controlling create/update/replace/destroy behavior.
Terraform lifecycle: what does `create_before_destroy` do?	Creates the replacement resource before destroying the current one.
Terraform lifecycle: what does `prevent_destroy` do?	Makes Terraform error if a plan would destroy the resource.
Does `prevent_destroy` protect a resource if you remove its configuration (Terraform)?	No; removing the block removes the rule, so Terraform can plan to destroy it.
Terraform lifecycle: what does `ignore_changes` do?	Tells Terraform to ignore diffs for listed attributes and not try to update them.
Terraform lifecycle: what does `replace_triggered_by` do?	Forces replacement when the referenced resources/attributes change.
Terraform lifecycle: what is a `precondition`?	A condition that must be true before creating the resource; otherwise Terraform errors with `error_message`.
Terraform lifecycle: what is a `postcondition`?	A condition checked after creating the resource; otherwise Terraform errors with `error_message`.
Terraform language: what is an expression?	Syntax that produces a value in configuration (e.g., literals, references, operators, conditionals, function calls).
What is a reference expression (Terraform)?	An expression that reads a value exported elsewhere (e.g., `var.<name>` or `<TYPE>.<NAME>.<ATTRIBUTE>`).
What command opens Terraform’s expression console?	`terraform console`
What is `terraform console` used for (Terraform)?	To experiment with and evaluate expressions interactively.
Terraform type: what is `string`?	Unicode text, like `"hello"`.
Terraform type: what is `number`?	A numeric value (can represent whole numbers and fractional values).
Terraform type: what is `bool`?	A boolean value: `true` or `false`.
Terraform type: what is `list`/`tuple` (high level)?	An ordered sequence you index with whole numbers starting at 0 (e.g., `["a", 15, true]`).
Terraform type: what is `set` (high level)?	An unordered collection of unique values (no stable index ordering).
Terraform type: what is `map`/`object` (high level)?	Values identified by string keys/labels, like `{ name = "Mabel", age = 52 }`.
Terraform types: what are “primitive” vs “complex/collection” types?	Primitive: `string`, `number`, `bool`; complex/collection: lists/tuples, maps/objects (and `set` as a collection type).
What special value has no type in Terraform?	`null`.
What does `null` mean in Terraform configuration?	Absence/omission: setting an argument to `null` behaves as if the argument were omitted.
If you set an argument to `null`, what does Terraform do with defaults/required args?	It uses the argument’s default if it has one; otherwise it errors if the argument is required.
Why is `null` especially useful in Terraform expressions?	To conditionally omit an argument when a condition isn’t met.
Terraform literal: how do you write a list/tuple literal?	`[value1, value2, ...]` (commas required; multiline allowed; trailing comma allowed).
Terraform literal: how do you write a map/object literal?	`{ key = value, ... }` (pairs can be separated by commas or line breaks).
How do you access list/tuple elements vs map/object elements in Terraform expressions?	`seq[3]` uses a whole-number index; `map["key"]` uses a string key.
When can you use dot-attribute notation on a map/object (Terraform)?	When the attribute name is a valid identifier (e.g., `local.object.attrname`).
Terraform type conversion: when does Terraform automatically convert types?	When an argument expects a type and the given expression can be converted to it.
Terraform strings: what is a heredoc string?	A multiline string literal introduced with `<<DELIM` (or `<<-DELIM`) and ended by `DELIM` on its own line.
Terraform heredoc: what does `<<-` mean (indented heredoc)?	Terraform trims common leading spaces from all lines (based on the least-indented line in the heredoc).
Terraform tip: should you use heredocs to generate JSON/YAML?	No; prefer `jsonencode(...)` / `yamlencode(...)` to guarantee valid syntax.
Terraform interpolation: what does `${ ... }` do?	Evaluates the expression, converts the result to a string if needed, and inserts it into the string.
Terraform directives: what does `%{ ... }` enable?	Conditionals and iteration over collections to build strings.
Terraform directive: what is the `%{if ...}` / `%{else}` / `%{endif}` pattern?	Choose between templates based on a boolean expression (the else is optional).
Terraform directive: what is the `%{for name in collection}` / `%{endfor}` pattern?	Iterate over elements and concatenate the rendered template for each element.
Terraform templates: what is the strip marker `~` for?	Whitespace stripping: it consumes literal whitespace (spaces/newlines) before or after a template sequence to avoid unwanted output.
Terraform style: when should you prefer heredocs vs quoted strings for templates?	Use heredocs for directives (multi-line templates); quoted strings are usually best for simple interpolations.
Terraform named values: what are the main categories?	Resources; input variables; local values; child module outputs; data sources; filesystem/workspace info; block-local values.
Terraform resource reference: what does `<TYPE>.<NAME>` refer to?	A managed resource instance (or instances) of the given type and local name.
If a resource uses neither `count` nor `for_each`, what is the type/shape of `<TYPE>.<NAME>`?	An object representing the resource, with attributes you can access (e.g., `<TYPE>.<NAME>.id`).
If a resource uses `count`, what is the type/shape of `<TYPE>.<NAME>`?	A list of instance objects.
If a resource uses `for_each`, what is the type/shape of `<TYPE>.<NAME>`?	A map of instance objects keyed by the `for_each` keys.
Terraform variables: what does `var.<NAME>` refer to?	The value of the input variable named `<NAME>`.
Terraform locals: what does `local.<NAME>` refer to?	The value of the local value named `<NAME>`.
Can locals reference other locals in the same `locals {}` block (Terraform)?	Yes, as long as you don’t create circular dependencies.
Terraform child module: what does `module.<MODULE_NAME>` represent?	The result of a `module` block (its outputs exposed as a value).
If a `module` block uses neither `count` nor `for_each`, what is `module.<MODULE_NAME>`?	An object with one attribute per child-module output (access via `module.<MODULE_NAME>.<OUTPUT_NAME>`).
If a `module` block uses `for_each`, what is `module.<MODULE_NAME>`?	A map of objects keyed by the `for_each` keys, each object holding the module outputs.
If a `module` block uses `count`, what is `module.<MODULE_NAME>`?	A list of objects, one per module instance, each object holding the module outputs.
Terraform data sources: what does `data.<DATA_TYPE>.<NAME>` represent?	An object (or instances) for a data resource of that type and name.
Terraform paths: what is `path.module`?	The filesystem path of the module where the expression appears.
Terraform paths: what is `path.root`?	The filesystem path of the root module of the configuration.
Terraform paths: what is `path.cwd`?	The original working directory where Terraform was run (before any `-chdir`).
Why avoid `path.module` in write operations (Terraform)?	Local module calls can share the same source directory, so writes can clobber each other and create race conditions.
Terraform workspaces: what is `terraform.workspace`?	The name of the currently selected workspace.
Why can using `terraform.workspace` for namespacing break module composability?	It can prevent calling the same module more than once in a configuration (name collisions/assumptions).
Block-local values: what is `count.index`?	The per-instance index available inside blocks that use `count`.
Block-local values: what are `each.key` and `each.value`?	The per-instance key and value available inside blocks that use `for_each`.
Block-local values: what is `self` used for?	A reference to the current resource in provisioner/connection blocks.
With `count`, how do you get an attribute of the first instance (Terraform)?	Index the list: `<TYPE>.<NAME>[0].id` (or similar).
With `count`, how do you get a list of an attribute across instances (Terraform)?	Use splat: `<TYPE>.<NAME>[*].id`.
With `for_each`, how do you access a particular instance attribute (Terraform)?	Index by key: `<TYPE>.<NAME>["key"].id`.
With `for_each`, how do you collect an attribute across instances (Terraform)?	Use a for-expression, e.g. `[for v in <TYPE>.<NAME> : v.id]`.
Terraform sensitive attributes: what happens in plan output?	Terraform shows a placeholder (like `(sensitive value)`) instead of the actual value.
If you derive a value from a sensitive resource attribute, how does Terraform treat it?	It typically marks derived values as sensitive too (so they’re also hidden in plan/apply output).
If an output depends on a sensitive value, what must you do (Terraform)?	Mark the output itself as `sensitive = true` to confirm you intend to export it.
Terraform unknown values: what are they (high level)?	Placeholders for values Terraform can’t know during planning (until apply).
How do unknown values propagate in expressions (Terraform)?	Operations that combine known + unknown usually produce unknown results.
Why can’t `count` be unknown (Terraform)?	Terraform must know the instance count during planning to build the graph and addresses.
What happens if a data source depends on unknown values (Terraform)?	It can’t be read during plan, so it’s deferred to apply and its results are unknown during planning.
When might type errors involving unknown values be detected (Terraform)?	Sometimes only at apply time, because full validation may be deferred until values are known.
What is an operator (Terraform language)?	An expression form that transforms one value or combines two values to produce a result.
How do binary operators appear in Terraform expressions?	Between two values (infix), e.g. `1 + 2`.
How do unary operators appear in Terraform expressions?	Before a value (prefix), e.g. `!true` or `-1`.
Terraform operator precedence: what is the order from highest to lowest?	`!` and unary `-`; `*` `/` `%`; `+` `-`; `>` `>=` `<` `<=`; `==` `!=`; `&&`; `||`.
How do you override operator precedence in Terraform?	Use parentheses `(...)`.
Without parentheses, how does Terraform parse `1 + 2 * 3`?	As `1 + (2 * 3)`.
In Terraform, are `?` and `:` considered operators?	No; together they are part of the conditional expression syntax.
What types do Terraform arithmetic operators expect?	`number` values (and they produce `number` results).
What does `a + b` do (Terraform)?	Adds `a` and `b`.
What does `a - b` do (Terraform)?	Subtracts `b` from `a`.
What does `a * b` do (Terraform)?	Multiplies `a` and `b`.
What does `a / b` do (Terraform)?	Divides `a` by `b`.
What does `a % b` do (Terraform)?	Returns the remainder of dividing `a` by `b` (most useful with whole numbers).
What does unary `-a` do (Terraform)?	Negates `a` (multiplies by `-1`).
How do you compute exponents in Terraform (no operator)?	Use `pow(base, exponent)`.
What types do Terraform equality operators expect and return?	They accept any types and return `bool`.
What does `a == b` mean in Terraform?	True only if `a` and `b` have the same type and the same value.
What does `a != b` mean in Terraform?	The negation of `a == b`.
Terraform best practice: when should you use `==`/`!=`?	Prefer primitive values, or use explicit type conversions to make types match.
Why can equality on structural types be surprising in Terraform?	Different structural types can look similar but never compare equal (e.g., an empty tuple literal `[]` vs an empty list value).
Terraform idiom: how to check whether a list is empty?	Use `length(var.list) == 0` instead of `var.list == []`.
What types do Terraform comparison operators expect and return?	They expect `number` values and return `bool`.
What does `a < b` / `a <= b` / `a > b` / `a >= b` do (Terraform)?	Performs numeric comparisons and yields a boolean.
What types do Terraform logical operators expect and return?	They expect `bool` values and return `bool`.
What does `a || b` do (Terraform)?	True if either operand is true; false if both are false.
What does `a && b` do (Terraform)?	True if both operands are true; false otherwise.
What does `!a` do (Terraform)?	Logical NOT: true if `a` is false; false if `a` is true.
Does Terraform have an XOR operator?	No; for boolean values, XOR is equivalent to `a != b`.
How does Terraform handle wrong operand types for operators (high level)?	It tries automatic conversion to the required types; otherwise it produces an error.
Terraform: what are built-in functions used for in expressions?	To transform and combine values; they’re called with function-call syntax (see function reference for the full list).
What is the general function call syntax in Terraform?	`FUNCTION_NAME(arg1, arg2, ...)`.
What does each Terraform function define about its interface?	Expected argument count/types and the return value type.
What does a Terraform function call expression evaluate to?	The function’s return value.
Can Terraform functions take a variable number of arguments?	Yes; e.g. `min(55, 3453, 2)` returns the smallest number.
How do you pass a list/tuple as separate function arguments in Terraform?	Use argument expansion with `...`, e.g. `min([55, 2453, 2]...)`.
In Terraform, what exactly is the expansion symbol for function arguments?	Three ASCII periods `...` (not a Unicode ellipsis `…`).
Where is `...` argument expansion allowed in Terraform?	Only in function calls (special syntax for expanding list/tuple arguments).
What happens if you pass sensitive data into a Terraform function call?	The result is marked sensitive (conservatively, regardless of which function is called).
Are most Terraform built-in functions “pure” (programming sense)?	Yes; results depend only on arguments, so call timing generally doesn’t matter.
Name four special Terraform functions whose timing can matter.	`file`, `templatefile`, `timestamp`, `uuid`.
When does Terraform evaluate `file(...)` and `templatefile(...)` (high level)?	During initial configuration validation, before planning/applying actions.
What practical limitation follows from `file(...)`/`templatefile(...)` being evaluated during validation?	You can’t use them to read files generated dynamically during plan/apply.
How do `timestamp()` and `uuid()` behave during plan vs apply (Terraform)?	They produce unknown values during plan; the real values are decided during apply (timestamp corresponds to when apply begins).
Terraform: what is a conditional expression used for?	Select one of two values based on a boolean condition.
What is the syntax of a conditional expression (Terraform)?	`condition ? true_val : false_val`.
How does `condition ? a : b` evaluate (Terraform)?	If `condition` is true, result is `a`; otherwise result is `b`.
Common use: how do conditional expressions define defaults (Terraform)?	Example: `var.a == "" ? "default-a" : var.a`.
What types of expressions usually appear in the condition part (Terraform)?	Any expression that evaluates to `bool` (often equality/comparison/logical operators).
Why add custom conditions in Terraform configs (high level)?	To capture assumptions and produce earlier, clearer error messages (e.g., in variable validation or pre/postconditions).
Conditional expressions: what constraint applies to the two result values (Terraform)?	They must have the same type, so Terraform can determine the expression’s result type.
If the two conditional result expressions have different types, what does Terraform do (Terraform)?	It attempts to find a type both can convert to and applies automatic conversions if possible.
Terraform recommendation: what should you do if the expected result type may be unclear?	Use explicit type conversion functions (e.g., `tostring(...)`) rather than relying on implicit conversion.
Example: how do you explicitly force a conditional to return a string (Terraform)?	`var.example ? tostring(12) : "hello"`.
Validation helper: how do you check membership in an allowed set (Terraform)?	`contains(["STAGE", "PROD"], var.environment)`.
Validation helper: how do you require a non-empty list/map (Terraform)?	`length(var.items) != 0`.
Why prefer `length(...)` checks over `== []` / `!= []` for empty collections (Terraform)?	Empty collection literals can be ambiguous in type; length-based checks express intent and avoid type-mismatch surprises.
Collection-wide validation pattern: how do you check a condition holds for all elements (Terraform)?	Use a for-expression + `alltrue([...])`, e.g. `alltrue([for v in var.instances : <bool expr>])`.
Collection-wide validation pattern: how do you check a condition holds for any element (Terraform)?	Use a for-expression + `anytrue([...])`.
What does `can(expr)` do in Terraform conditions?	Returns true if `expr` evaluates without error; false if evaluating it produces an error.
Example: how can `can` be used to validate a string with `regex` (Terraform)?	`can(regex("^[a-z]+$", var.name))` (because `regex` errors on non-match).
In conditions, what are `self`, `each`, and `count` used for (Terraform)?	`self` refers to attributes of the instance under evaluation in postcondition blocks; `each`/`count` refer to per-instance values inside blocks using `for_each`/`count`.
What does a for expression do (Terraform)?	A for expression transforms one collection into another by evaluating an expression per element (optionally filtering elements out).
Terraform for expression: what input types are allowed after `in`?	List, set, tuple, map, or object.
Terraform for expression: what does `[for s in var.list : upper(s)]` produce?	A tuple of results in the same order as the input collection (after any implied ordering rules).
Terraform for expression: what does the single symbol after `for` represent?	The element value for each item in the input collection.
Terraform for expression: what does the two-symbol form `for k, v in ...` provide?	Two temporary symbols: key/index (`k`) and value (`v`) for each element.
In a Terraform for expression over a list/tuple, what is the first symbol in `for i, v in ...`?	The zero-based index of each element.
In a Terraform for expression over a map/object, what is the first symbol in `for k, v in ...`?	The key (map key) or attribute name (object attribute).
Terraform for expression: what result type do `[...]` brackets produce?	A tuple.
Terraform for expression: what result type do `{...}` braces produce, and what syntax is required?	An object; you must provide `key_expr => value_expr`.
Terraform for expression: how do you filter elements?	Add `if <condition>` at the end (e.g., `[for s in var.list : upper(s) if s != \"\"]`).
Terraform for expression: why might you split one collection into two with `if`?	To build separate collections by predicate (e.g., admins vs regular users).
Terraform for expressions: what is the implied ordering for maps/objects when converting to an ordered result?	Sorted lexically by key/attribute name.
Terraform for expressions: what is the implied ordering for a set of strings when converting to an ordered result?	Sorted lexically by string value.
Terraform for expressions: what ordering is used for sets of non-strings?	Arbitrary ordering that may change between Terraform versions.
Terraform for expressions: why convert a result to a set with `toset(...)` sometimes?	To make it explicit that the result is unordered (especially when the source is an unordered collection).
Terraform for expressions: what is “grouping mode” in object comprehensions?	Add `...` after the value expression to allow duplicate keys, producing a map of lists (multiple values per key).
Terraform grouping mode: what does `role => name...` mean?	Group multiple `name` values under the same `role` key (value becomes a list of strings per role).
Can Terraform for expressions generate nested blocks in resources?	No; for nested blocks use `dynamic` blocks (for expressions only construct values inside expressions).
What is a splat expression for (Terraform)?	A concise way to map over a collection and extract nested attributes/indexes (often equivalent to a for-expression).
Splat vs for-expression: what does `var.list[*].id` correspond to (Terraform)?	`[for o in var.list : o.id]`.
What does the `[*]` operator do in a splat expression (Terraform)?	Iterates over all elements on the left and applies the access operations on the right to each element.
Can splat expressions chain deeper attribute/index accesses (Terraform)?	Yes; e.g. `var.list[*].interfaces[0].name` applies `.interfaces[0].name` to each element.
What collection types support splat expressions (Terraform)?	Lists, sets, and tuples.
How do you get a “splat-like” result for a map/object (Terraform)?	Use a for-expression (splats don’t work on maps/objects).
Why don’t splat expressions work on `for_each` resources (Terraform)?	`for_each` resources are maps of objects; splats require list/set/tuple.
How do you get a list of attributes from a `for_each` resource anyway (Terraform)?	Convert to a list with `values(...)` and then splat, e.g. `values(aws_x.y)[*].id`.
Splat special case: what happens if you apply `[*]` to a non-collection, non-null value (Terraform)?	It becomes a single-element tuple (a “single value as list” behavior).
Splat special case: what does `[*]` return if the value is `null` (Terraform)?	An empty tuple.
Common use: why use `var.obj[*]` for an optional object input (Terraform)?	To drive `for_each` in a `dynamic` block: 0 blocks if `null`, 1 block if set.
Terraform guidance: where is the “single value as list” splat trick recommended (Terraform)?	In `for_each`-style contexts where a collection is expected; otherwise it can confuse readers.
What is the legacy “attribute-only” splat syntax (Terraform)?	Use `.*.` (e.g., `var.list.*.attr`) instead of modern `[*]`.
Why is legacy `.*` splat discouraged (Terraform)?	Its behavior is less consistent and can be surprising; prefer modern `[*]`.
Legacy splat gotcha: how does indexing differ between `.*` and `[*]` (Terraform)?	In `.*`, indexes like `[0]` apply after the iteration result rather than per-element, unlike `[*]`.
Why do you need `dynamic` blocks (Terraform)?	Some resource arguments are repeatable nested blocks that can’t be generated with expressions in `name = expr` form; `dynamic` generates nested blocks from data.
What does a `dynamic` block produce (Terraform)?	Repeatable nested blocks (not a collection value), one nested block per element of the `for_each` collection.
Where can you use `dynamic` blocks (Terraform)?	Inside `resource`, `data`, `provider`, and `provisioner` blocks.
What does the label of a `dynamic` block mean (Terraform)?	The nested block type name to generate (e.g., `dynamic "setting"` generates `setting { ... }` blocks).
In a `dynamic` block, what does `for_each` do (Terraform)?	Supplies the collection/structural value to iterate over; one nested block is generated per element.
In a `dynamic` block, what does `content { ... }` define (Terraform)?	The body of each generated nested block.
In a `dynamic` block, what does `iterator` control (Terraform)?	The name of the temporary variable for the current element; defaults to the dynamic label if omitted.
What are `iterator.key` and `iterator.value` in a `dynamic` block (Terraform)?	`key` is map key or list index (for sets, `key == value` and shouldn’t be used); `value` is the element value.
In a `dynamic` block iterating a set, what’s special about `.key` (Terraform)?	It’s identical to `.value`, so it’s not useful and should generally not be used.
In a `dynamic` block, what does `labels` do (Terraform)?	Optional list of strings to set the generated block labels (in order); can use the iterator variable to compute labels.
Can `dynamic` blocks generate meta-argument blocks like `lifecycle` (Terraform)?	No; they can only generate blocks belonging to the configured type (resource/data/provider/provisioner), not meta-argument blocks.
Why can’t `dynamic` generate `lifecycle`/`provisioner` meta-argument blocks (Terraform)?	Terraform must process meta-arguments before it is safe to evaluate expressions that would generate them.
Can you nest `dynamic` blocks (Terraform)?	Yes; you can generate multi-level nested block structures by putting `dynamic` blocks inside the `content` of outer ones.
Nested dynamic blocks: what iterator pitfall should you watch for (Terraform)?	Inner and outer iterators differ (e.g., `origin_group.value` vs `origin.value`); choose iterator names carefully to avoid confusion.
When should you set `iterator` explicitly in nested `dynamic` blocks (Terraform)?	When nested block type names collide or readability suffers; pick distinct iterator symbols.
Best practice: when should you avoid `dynamic` blocks (Terraform)?	When you can write nested blocks literally; overuse hurts readability and maintainability.
What’s a sign your module abstraction may be weak (Terraform `dynamic` blocks)?	If most resource arguments/blocks are just passed through from an input variable, the caller may be better off defining the resource directly.
How can `dynamic` `for_each` use `for`/splat expressions (Terraform)?	Because `for_each` accepts any collection/structural value, you can transform data first with `for` expressions or splats and then iterate it.
